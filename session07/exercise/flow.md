# 実習: リクエスト/レスポンスの流れを書いてみよう

TODO アプリで「新しい TODO を追加する」とき、ブラウザとサーバーの間で
何が起きているのかを順序立てて書いてみましょう。

コードを読むだけではなく、自分の言葉で説明することで
「フロントエンド」「バックエンド」「データベース」の役割分担が見えてきます。

---

## シナリオ

> ユーザーがブラウザ上で「牛乳を買う」と入力して**追加**ボタンを押した。
> 画面のリストに「牛乳を買う」が表示された。
>
> このとき、ブラウザとサーバーの間では何が起きている？

---

## 書き方のヒント

以下の登場人物を使って、「誰が」「何を」「どこへ」するのかを意識して書きましょう。

- **ブラウザ**（HTML / CSS / JavaScript が動く場所）
  - `static/app.js` の関数
  - `fetch()`
- **サーバー**（FastAPI = `main.py` が動く場所）
  - `@app.post("/todos")` のエンドポイント
- **データベース**（`todo.db`）
  - `INSERT INTO todos ...`

## 確認ポイント

書き終わったら以下をチェックしてみよう:

- [ ] 「ブラウザ」と「サーバー」のどちらが動いているのかが、各ステップで分かるように書けているか
- [ ] `fetch()` がどのタイミングで呼ばれるかを書けているか
- [ ] リクエストの **method**（GET / POST / PUT / DELETE）と **URL** を書けているか
- [ ] サーバーがデータベースに何をしているか（SELECT / INSERT / UPDATE / DELETE）を書けているか
- [ ] レスポンスがブラウザに戻ってきたあと、画面がどう更新されるかを書けているか
